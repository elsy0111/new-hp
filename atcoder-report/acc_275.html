A - Find Takahashi AC

- 問題 -

AtCoder 村には $N$ 本の橋があり、$i$ 本目（ $1\leq i\leq N$ , $i \in \mathbb{R}$ ） の橋の高さは $H_i$です。
ここで、AtCoder 村にある N 本の橋のうち、どの相異なる $2$ 本の橋も高さが異なります。
AtCoder 村で最も高い橋は何本目の橋か出力してください。

制約

$1\leq N\leq 100$
$1\leq H_i \leq 10^9$
$H_i$ は全て異なる
$N,H \in \mathbb{Z}$

入力

入力は以下の形式で標準入力から与えられる。

$N$
$H_1\quad H_2\quad ...\quad H_N$

出力

AtCoder 村で最も高い橋は何本目の橋かを、整数で出力せよ。

入力例

3
50 80 70

出力例

2

- 解説 -

問題文の要約

$max(h)$ のインデックス。
$\large \Rightarrow \overset{\Large argmax(h)}{\small i \quad}$
便利なことに、この $argmax$ をそのまま実装できるライブラリが numpy にあります。
それを使って実装してみましょう。

Code

import numpy as np

n = int(input())
h = list(map(int,input().split()))

print(np.argmax(h) + 1)

ほかの言語がかわいそうになるくらい簡潔に実装できましたね...

B - ABC-DEF AC

- 問題 -

非負整数 $A,B,C,D,E,F$ があり、$A×B×C\geq D×E×F$ をみたしています。
$(A×B×C)−(D×E×F)$ の値を $998244353$ で割った余りを求めてください。

制約

$1\leq A,B,C,D,E,F\leq 10^{18}$
$A×B×C \geq D×E×F$
$A,B,C,D,E,F \in \mathbb{Z}$

入力

入力は以下の形式で標準入力から与えられる。

$A\quad B\quad C\quad D\quad E\quad F\quad$

出力

$(A×B×C)−(D×E×F)$ の値を $998244353$ で割った余りを整数で出力せよ。

入力例

2 3 5 1 2 4

出力例

22

- 解説 -

問題文の要約

圧倒的 B<<A。
要約も何もないです。

Code

a,b,c,d,e,f = list(map(int,input().split()))

print(((a*b*c)-(d*e*f))%998244353)
C - Counting Squares WA
D - Robot Arms 2 TLE

- 問題 -

長さ $N$ の正整数列 $A=(A_1,A_2,...,A_N)$ および整数 $x,y$ が与えられます。
次の条件をすべて満たすように、$xy$ 座標平面上に $N+1$ 個の 点 $p_1,p_2,...,p_N,p_{N+1}$ を配置することができるか判定してください。 (同じ座標に $2$ 個以上の点を配置してもよいです。)

$1) \quad p_1 = (0,0)$
$2) \quad p_2 = (A_1,0)$
$3) \quad$点 $p_i$ と点 $p_{i+1}$ の距離は $A_i\,(1\leq i\leq N)$
$4) \quad$線分 $p_i \, p_{i+1}$ と 線分 $p_{i+1} \, p_{i+2}$ のなす角は $90$ 度 $\,(1\leq i\leq N-1)$

制約

$2\leq N\leq 10^3$
$1\leq A_i\leq 10$
$|x|,|y|\leq 10^4$
$N,A_i,x,y \in \mathbb{Z}$

入力

入力は以下の形式で標準入力から与えられる。

$N\quad x\quad y$
$A_1\quad A_2\quad ... \quad A_N\quad$

出力

問題文の条件をすべて満たすように $p_1,p_2,...,p_N,p_{N+1}$ を配置することができる場合は Yes を、 そうでない場合は No を出力せよ。

Yes
No

入力例

3 -1 1
2 1 3

出力例

Yes

- 解説 -

問題文の要約

条件 $1),2)$より、( $p_2$ は固定なので) 最初 $p_2 \to p_3$ は $(p_2,0)$ から $y$ 軸方向へ進むことがわかる。
条件 $4)$ より、 $x$ 軸, $y$ 軸方向へ交互に $90$ 度ずつ、ジグザグに進むことがわかる。
したがって、問題文は以下のように訳すことができる。

$\alpha : \: $$x \,$軸方向において $A_2$ に $A_i \,$( $i$ は偶数) を足すか引くかして $x$ になるか
$\beta : \: $$y \,$軸方向において $0$ に $A_i \,$( $i$ は奇数) を足すか引くかして $y$ になるか
$\alpha \: \land \beta$ を満たすとき、$p_2$ から $p_{N+1}$ まで移動できたといえる。

Code

None

言い訳

この問題をそのまま解こうとすると足すか引くかの $2$ 通りを全て調べる必要があります(全探索)。
なので、計算量としては $O(2^N)$ となり、最悪の場合 $10^{301}$(回) ほどの計算が必要になります。
ですのでむりです。
