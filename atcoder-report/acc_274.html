A - Batting Average AC

- 問題 -

整数 $A,B$ があります。ここで $A,B$ は $1\leq A\leq 10 \: ,\: 0\leq B\leq A$ を満たします。 このとき、文字列 $S$ を次のように定義します。

${\Large \frac{B}{A}}$ を小数点第 $4$ 位で四捨五入した値を「整数部 $1$ 桁」「 . 」「小数部 $3$ 桁」の順に末尾ゼロを省略せずに表記した文字列。

例えば $A=7,B=4$ の場合は、${\Large \frac{B}{A}} = {\Large \frac{4}{7}} = 0.571428…$ で、これを小数点第 $4$ 位で四捨五入した値は $0.571$です。 よって $S$ は $0.571$ になります。

$A,B$ が入力として与えられるので $S$ を出力してください。

制約

$1\leq A\leq 10$
$0\leq B \leq A$
$A,B \in \mathbb{Z}$

入力

入力は以下の形式で標準入力から与えられる。

$A\quad B$

出力

$S$ を問題文の指示に従った形式で出力せよ。

入力例

7 4

出力例

0.571

- 解説 -

問題文の要約

小数第四位を四捨五入して第三桁目まで出力してください。但し、出力する形式は小数第三位まで埋めてください。

Code

a,b = map(int,input().split())

x = b/a
y = round(x, 3)

print('{:.3f}'.format(y))
B - Line Sensor AC

- 問題 -

縦 $H$ マス、横 $W$ マスのグリッドがあります。上から $i$ 行目、左から $j$ 列目のマスを $(i,j)$ と表します。 各マスの状態は文字 $C_{i,j}$ で表されます。 $C_{i,j}$ が . ならば $(i,j)$ には何も置かれておらず、 # ならば箱が $1$ 個置かれています。
$1\leq j\leq W$ を満たす整数 $j$ に対して、整数 $X_j$ を次のように定義します。

$j$ 列目に置かれている箱の個数。言い換えると、$C_{i,j}$ が # であるような整数 $i$ の個数。

$X_1,X_2,...,X_W$ をすべて求めてください。

制約

$1\leq H\leq 1000$
$1\leq W\leq 1000$
$H,W \in \mathbb{Z}$
$C_{i,j} \in \{\hspace{7px} .,\verb|#| \hspace{7px} \}$

入力

入力は以下の形式で標準入力から与えられる。

$H\quad W$
$C_{1,1}C_{1,2} ... C_{1,W}$
$C_{2,1}C_{2,2} ... C_{2,W}$
$\hspace{7px} \vdots$
$C_{H,1}C_{H,2} ... C_{H,W}$

出力

$X_1,X_2,...,X_W$ を以下の形式に従って出力せよ。

$X_1\quad X_2\quad ...\quad X_W$

入力例

3 4
#..#
.#.#
.#.#

出力例

1 2 0 3

- 解説 -

問題文の要約

縦横のテーブルの縦(列)に、#は何個あるか。

$\Rightarrow$ 二次元配列を作り、各行の同じ個数目の要素に#は何個あるか。

Code

h,w = map(int,input().split())

//arr : 縦h行*横w列の二次元配列
arr = []
for i in range(h):
    arr.append(input())

ans = []                //解答の格納
for j in range(w):      //全(w)列回す
    cnt = 0
    for i in range(h):  //全(h)行回す
        if arr[i][j] == '#':
            cnt += 1
    ans.append(cnt)

//出力
for i in ans[:len(ans) - 1]:
    print(i,end = " ")
print(ans[-1])

D - Robot Arms 2 TLE

- 問題 -

長さ $N$ の正整数列 $A=(A_1,A_2,...,A_N)$ および整数 $x,y$ が与えられます。
次の条件をすべて満たすように、$xy$ 座標平面上に $N+1$ 個の 点 $p_1,p_2,...,p_N,p_{N+1}$ を配置することができるか判定してください。 (同じ座標に $2$ 個以上の点を配置してもよいです。)

$1) \quad p_1 = (0,0)$
$2) \quad p_2 = (A_1,0)$
$3) \quad$点 $p_i$ と点 $p_{i+1}$ の距離は $A_i\,(1\leq i\leq N)$
$4) \quad$線分 $p_i \, p_{i+1}$ と 線分 $p_{i+1} \, p_{i+2}$ のなす角は $90$ 度 $\,(1\leq i\leq N-1)$

制約

$2\leq N\leq 10^3$
$1\leq A_i\leq 10$
$|x|,|y|\leq 10^4$
$N,A_i,x,y \in \mathbb{Z}$

入力

入力は以下の形式で標準入力から与えられる。

$N\quad x\quad y$
$A_1\quad A_2\quad ... \quad A_N\quad$

出力

問題文の条件をすべて満たすように $p_1,p_2,...,p_N,p_{N+1}$ を配置することができる場合は Yes を、 そうでない場合は No を出力せよ。

Yes
No

入力例

3 -1 1
2 1 3

出力例

Yes

- 解説 -

問題文の要約

条件 $1),2)$より、( $p_2$ は固定なので) 最初 $p_2 \to p_3$ は $(p_2,0)$ から $y$ 軸方向へ進むことがわかる。
条件 $4)$ より、 $x$ 軸, $y$ 軸方向へ交互に $90$ 度ずつ、ジグザグに進むことがわかる。
したがって、問題文は以下のように訳すことができる。

$\alpha : \: $$x \,$軸方向において $A_2$ に $A_i \,$( $i$ は偶数) を足すか引くかして $x$ になるか
$\beta : \: $$y \,$軸方向において $0$ に $A_i \,$( $i$ は奇数) を足すか引くかして $y$ になるか
$\alpha \: \land \beta$ を満たすとき、$p_2$ から $p_{N+1}$ まで移動できたといえる。

Code

None

言い訳

この問題をそのまま解こうとすると足すか引くかの $2$ 通りをN個全て調べる必要があります(全探索)。
なので、計算量としては $O(2^N)$ となり、最悪の場合(制約より) $2^{10^3} = 10^{301}$(回) ほどの計算が必要になります。
ちなみにコンピュータが一秒間に処理できる回数は $10^8$~$10^9$です。
ですのでむりです。あきらめましょう。

実際はこのような問題に対しては 動的計画法 $dp$ というアルゴリズムを用いて 計算量を削減することができます。
私はその辺には疎いのでアルゴリズムの勉強をしてみます。

おわり。
